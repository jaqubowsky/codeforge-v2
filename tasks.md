## Milestone 3: Job Matching with Embeddings

**Goal:** Implement semantic job matching using local embeddings from Milestone 2.5.

**Prerequisites:** Milestone 2.5 must be complete (embeddings infrastructure set up).

### Tasks

- [ ] **3.1 Verify pgvector extension is enabled**
  ```sql
  CREATE EXTENSION IF NOT EXISTS vector;
  ```
  _(Note: Should already be done in profiles migration)_

- [ ] **3.2 Verify embedding columns exist with correct dimensions**
  - Profiles table: `embedding VECTOR(384)`
  - Offers table: `embedding VECTOR(384)`
  - Vector indexes created with `vector_cosine_ops`

- [ ] **3.3 Create similarity search function**
  ```sql
  CREATE FUNCTION match_jobs_for_user(
    user_embedding VECTOR(384),
    match_threshold FLOAT DEFAULT 0.7,
    match_count INT DEFAULT 50
  ) RETURNS TABLE (offer_id BIGINT, similarity FLOAT)
  LANGUAGE sql STABLE
  AS $$
    SELECT id, 1 - (embedding <=> user_embedding) AS similarity
    FROM public.offers
    WHERE embedding IS NOT NULL
      AND 1 - (embedding <=> user_embedding) >= match_threshold
    ORDER BY embedding <=> user_embedding
    LIMIT match_count;
  $$;
  ```
  - Uses cosine similarity (via `<=>` operator)
  - Returns jobs ordered by relevance
  - Filters by threshold

## Milestone 4: User-Job Relationships

  CREATE TYPE public.user_offer_status AS ENUM ('inbox', 'saved', 'archived');

  CREATE TABLE public.user_offers (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,

## Architecture Overview

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Next.js Web   │────▶│    Supabase      │◀────│  Cloud Functions│
│   (Vercel)      │     │  (Postgres +     │     │  (Scraper)      │
│                 │     │   pgvector)      │     │                 │
└─────────────────┘     └──────────────────┘     └─────────────────┘
        │                       │                        │
        │                       │                        │
        ▼                       ▼                        ▼
   User actions          Embeddings storage        Job scraping +
   Profile mgmt          Similarity search         Embedding gen
   Job display           User data (RLS)
```

**Embedding Flow:**
1. Scraper fetches job → Generates embedding → Stores in `offers.embedding`
2. User completes onboarding → Generates embedding → Stores in `profiles.embedding`
3. User clicks "Find Jobs" → Cosine similarity search → Results in `user_offers`
