## Milestone 3: Embeddings Infrastructure

**Goal:** Set up pgvector and embedding generation for semantic job matching.

### Tasks

- [ ] **3.1 Enable pgvector extension in Supabase**
  ```sql
  CREATE EXTENSION IF NOT EXISTS vector;
  ```

- [ ] **3.2 Add embedding column to offers table (migration)**
  ```sql
  ALTER TABLE public.offers ADD COLUMN embedding VECTOR(1536);
  CREATE INDEX offers_embedding_idx ON public.offers
    USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
  ```

- [ ] **3.3 Create embedding generation service**
  - Use OpenAI `text-embedding-ada-002` or `text-embedding-3-small`
  - Create shared function in database package or scraper
  - Input: text string → Output: 1536-dimension vector

- [ ] **3.4 Create similarity search function**
  ```sql
  CREATE FUNCTION match_jobs_for_user(
    user_embedding VECTOR(1536),
    match_threshold FLOAT DEFAULT 0.7,
    match_count INT DEFAULT 50
  ) RETURNS TABLE (offer_id BIGINT, similarity FLOAT)
  ```
  - Return jobs ordered by cosine similarity
  - Filter by threshold

## Milestone 4: User-Job Relationships

**Goal:** Track which jobs users have seen, saved, or archived.

### Tasks

- [ ] **4.1 Create user_offers table (migration)**
  ```sql
  CREATE TYPE public.user_offer_status AS ENUM ('inbox', 'saved', 'archived');

  CREATE TABLE public.user_offers (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    offer_id BIGINT NOT NULL REFERENCES public.offers(id) ON DELETE CASCADE,
    status public.user_offer_status NOT NULL DEFAULT 'inbox',
    similarity_score FLOAT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id, offer_id)
  );
  ```
  - Add RLS policies
  - Add indexes on user_id, status

- [ ] **4.2 Create database queries in packages/database**
  - `getUserOffers(userId, status)` - get offers with status filter
  - `updateOfferStatus(userId, offerId, status)` - save/archive
  - `getMatchingJobs(userId)` - embedding similarity search

## Milestone 5: Scraper Enhancement

**Goal:** Update scraper to generate embeddings for new jobs.

### Tasks

- [ ] **5.1 Review existing scraper architecture**
  - Document current JustJoinIT strategy flow
  - Identify where to add embedding generation

- [ ] **5.2 Add embedding generation to scraper pipeline**
  - After scraping job details, generate embedding from: `{title} | {skills} | {description}`
  - Store embedding with offer
  - Handle API rate limits for embedding calls

- [ ] **5.3 Backfill embeddings for existing offers**
  - Create one-time script to generate embeddings for existing offers
  - Run in batches to avoid rate limits

- [ ] **5.4 Create user-triggered job matching endpoint**
  - Cloud Function: `matchJobsForUser`
  - Input: user_id
  - Process:
    1. Get user's profile embedding
    2. Run similarity search against offers
    3. Insert new matches into user_offers with status='inbox'
    4. Return count of new matches
  - Target: <30 seconds response time

## Architecture Overview

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Next.js Web   │────▶│    Supabase      │◀────│  Cloud Functions│
│   (Vercel)      │     │  (Postgres +     │     │  (Scraper)      │
│                 │     │   pgvector)      │     │                 │
└─────────────────┘     └──────────────────┘     └─────────────────┘
        │                       │                        │
        │                       │                        │
        ▼                       ▼                        ▼
   User actions          Embeddings storage        Job scraping +
   Profile mgmt          Similarity search         Embedding gen
   Job display           User data (RLS)
```

**Embedding Flow:**
1. Scraper fetches job → Generates embedding → Stores in `offers.embedding`
2. User completes onboarding → Generates embedding → Stores in `profiles.embedding`
3. User clicks "Find Jobs" → Cosine similarity search → Results in `user_offers`
